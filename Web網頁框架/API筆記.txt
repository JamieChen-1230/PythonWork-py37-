- REST API：
    - 主要原則：
        - REST API 是依照「資源」來設計的，而資源是指可由用戶端存取、任何類型的物件、資料或服務。
            EX: https://xxx.com/orders/
        - 資源具有「識別碼」，用來識別該資源。(通常是用id)
            EX: https://xxx.com/orders/1/
        - REST API 會使用統一的介面，有助於讓用戶端與服務實作分離，並依照不同的HTTP請求(GET、POST、PUT、PATCH和DELETE)做出不同操作。
            EX: GET https://xxx.com/orders/  =>  獲取order列表
                POST https://xxx.com/orders/  =>  新增一筆order


- 認證(Authentication)和授權(Authorization)：
    - 比喻：
        1.情境一(登機)：
            我們登機時會需要機票和護照，
            機票相當於是登上飛機的門票，也就是說，持有機票就能擁有登機的權利，
            護照或身分證則是為了驗證你是本人的，也就是說，當驗證成功後就能證明你是這張機票的合法持有人，
            最後，你就可以帶著這張機票登上飛機。
            認證(Authentication)： 驗證護照或身分證
            授權(Authorization)： 給予你能登上飛機的權力
            憑證(Credentials)： 機票
        2.情境二(部落格)：
            我們要登入部落格需要帳號和密碼，
            當帳密正確之後就能登入部落格，也就是說，當成功登入後就能證明你是一個合法用戶，
            而部落格會根據你的身分發給你一個令牌，讓你能根據這令牌的權限做出一系列操作(EX:增刪文章)。
            認證(Authentication)： 驗證帳號和密碼
            授權(Authorization)： 給予你能增刪文章的權力
            憑證(Credentials)： 令牌
        3.情境三(問句)：
            認證(Authentication)： 我是誰？
            授權(Authorization)： 我能做什麼？
